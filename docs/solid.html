D · SOLID-Based Product Architecture

Eleven layers, five principles, one rule: domain logic never knows it's on Spark

The architecture has two orthogonal axes. Vertically: eleven processing layers, each mapped to a Databricks-native service. Horizontally: a SOLID decomposition that keeps domain logic, pipeline orchestration, and infrastructure strictly separated so a new domain or a new match algorithm is config or a plugin — never a fork.

Why a fourth medallion tier — the Master / Trust layer

Bronze/Silver/Gold is a refinement gradient: raw → cleaned → business-ready. It is silent on identity, which is the entire point of MDM. Cramming golden records into Gold conflates two unrelated concerns — "this table is curated for consumption" vs. "these rows are the surviving truth about a resolved entity with a trust score and a merge lineage." That conflation is why transplanted MDMs feel bolted-on.

TierQuestion it answersKey objectsGrain
BronzeWhat did the source say, verbatim?RawRecord1 row / source event (append-only)
SilverWhat does it mean, cleaned & validated?StandardizedRecord (+quality score)1 row / source record, conformed
Master / TrustWho is this, really, and how sure are we?Cluster, GoldenRecord, SurvivorAttribute(+trust), xref & merge lineage1 row / resolved entity + cluster membership
GoldHow do consumers want to see it?Domain marts, hierarchy views, serving projectionsConsumption-shaped (denormalized, joined)
Decision

Yes, justify a dedicated Master/Trust tier (physically a schema, e.g. mdm_<domain>.master). Gold is then a projection of Master for consumption — which keeps survivorship/lineage out of consumer marts and lets you re-shape Gold without re-resolving identity.

SOLID, in MDM terms

The governing rule for all five: domain logic (what a "match" or "survivor" means) must not import Spark, Delta, or any infrastructure. Pipelines orchestrate; infrastructure persists; domain decides. This is what makes the same match strategy testable on a pandas DataFrame in CI and runnable on a billion rows in production.

S · Single Responsibility

Meaning here: each component owns exactly one reason to change. Standardization changes when normalization rules change. Matching changes when match logic changes. They are not the same class.

Violation

The transplant's GoldenRecordBuilder that standardizes, matches, merges, and applies survivorship in one 22-KB module. A change to phone normalization risks breaking survivorship. Every edit is a regression-test of everything.

Pattern

Split into Standardizer, Matcher, Clusterer, GoldenRecordAssembler, SurvivorshipResolver — each a separate package with its own tests, wired together by an orchestration layer that owns no domain logic.

O · Open / Closed

Meaning here: add a new source, match algorithm, or survivorship rule without editing existing code — by registering a new strategy.

Violation

A hardcoded Random Forest inside the matcher and an if source == "SAP" ladder. Supporting probabilistic + semantic match means surgery on the matcher; adding a supplier domain means copy-pasting the customer pipeline.

Pattern

A MatchStrategy registry. DeterministicMatch, ProbabilisticMatch, VectorSemanticMatch, ZinggMatch all implement the same contract and are selected by config. New algorithm = new class + a YAML line.

L · Liskov Substitution

Meaning here: any strategy implementing a contract must be swappable without breaking the pipeline. A semantic matcher and a deterministic matcher must honour the same input/output schema and the same score semantics (0–1, higher = more confident).

Violation

One matcher returns booleans, another returns distances, a third returns labels. The clustering layer must special-case each — substitution is impossible and the abstraction is a lie.

Pattern

A typed MatchResult (candidate pair → calibrated probability + explanation). Every strategy returns it. Calibration (e.g. isotonic) makes scores comparable, so clustering treats all strategies uniformly.

I · Interface Segregation

Meaning here: a source connector shouldn't be forced to implement streaming if it's batch-only; a survivorship rule shouldn't depend on the publishing interface.

Violation

A fat Connector interface with readBatch, readStream, writeBack, schemaDrift, validate — so a simple CSV connector must stub five methods it never uses.

Pattern

Small role interfaces: BatchSource, StreamSource, CDCSource, SchemaProvider. A connector composes only the roles it supports.

D · Dependency Inversion

Meaning here — the keystone. High-level mastering policy depends on abstractions, not on Spark or Delta. Infrastructure is injected.

Violation

Survivorship logic calls spark.read.table(...) and df.write.format("delta") directly. Now you can't unit-test it without a cluster, can't reuse it for streaming, and can't port a rule into a serving path.

Pattern

Domain code receives a RecordRepository port; DeltaRecordRepository and InMemoryRecordRepository are adapters. The survivorship resolver is pure: records in, survivor out. Spark appears only in the adapter and the pipeline wiring.

The contracts (ports)

These nine interfaces are the product's spine. Domain logic depends only on these; Databricks-specific adapters implement them. Shown in Python-ish pseudocode for precision, not as a code deliverable.

# ---- ingestion -------------------------------------------------------------
class BatchSource(Protocol):
    def read(self, ctx: RunContext) -> RecordFrame: ...
class StreamSource(Protocol):
    def stream(self, ctx: RunContext) -> RecordStream: ...
class SchemaProvider(Protocol):
    def schema(self) -> CanonicalSchema: ...        # maps source → canonical

# ---- standardization -------------------------------------------------------
class Standardizer(Protocol):
    # pure: no Spark. operates on a domain Record, returns StandardizedRecord
    def standardize(self, r: Record, policy: StdPolicy) -> StandardizedRecord: ...
class QualityRule(Protocol):
    def evaluate(self, r: StandardizedRecord) -> QualityVerdict: ...   # score + flags

# ---- matching --------------------------------------------------------------
class BlockingStrategy(Protocol):
    def keys(self, r: StandardizedRecord) -> list[BlockKey]: ...
class MatchStrategy(Protocol):
    def score(self, a: StandardizedRecord, b: StandardizedRecord) -> MatchResult: ...
    # MatchResult = (prob: float[0..1], explanation: dict, strategy_id: str)

# ---- mastering -------------------------------------------------------------
class ClusterPolicy(Protocol):
    def cluster(self, pairs: Iterable[MatchResult]) -> Iterable[Cluster]: ...
class SurvivorshipRule(Protocol):
    def survive(self, attr: str, candidates: list[AttrValue],
                trust: TrustModel) -> SurvivorAttribute: ...   # +lineage

# ---- stewardship & workflow ------------------------------------------------
class StewardshipTask(Protocol):
    def open_case(self, cluster: Cluster, reason: str) -> StewardshipCase: ...
class ApprovalWorkflow(Protocol):
    def route(self, case: StewardshipCase) -> WorkflowState: ...     # state machine

# ---- publishing ------------------------------------------------------------
class PublicationAdapter(Protocol):
    def publish(self, events: Iterable[PublishEvent]) -> PublishReceipt: ...
    # impls: DeltaSharingAdapter, CDFAdapter, KafkaAdapter, JdbcAdapter, ReverseETLAdapter

# ---- the only Spark-aware boundary ----------------------------------------
class RecordRepository(Protocol):           # the injected port (DIP keystone)
    def load(self, ref: TableRef) -> RecordFrame: ...
    def upsert(self, ref: TableRef, frame: RecordFrame) -> None: ...
# DeltaRecordRepository  → real Spark/Delta
# InMemoryRecordRepository → pandas, used in CI unit tests
The separation, stated once

Domain (Standardizer, MatchStrategy, SurvivorshipRule, ClusterPolicy) = pure, Spark-free, unit-tested on pandas. Orchestration (Lakeflow/DLT graphs, Workflows) = wires ports together, owns no business rules. Infrastructure (DeltaRecordRepository, adapters, Vector Search client) = the only code that imports Spark/Delta/UC. Cross this wiring up and you get the transplant; honour it and a new domain is a config bundle, not a fork.
