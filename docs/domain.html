E · Domain-Driven Design & Canonical Model

Nine bounded contexts, one ubiquitous language, contracts at every seam

MDM fails when "customer" means six different things in six modules. DDD fixes that: each bounded context owns its model and vocabulary, contexts communicate only through explicit data contracts and domain events, and the canonical entity model is shared deliberately — not by accident.

Aggregates & consistency boundaries

  • Cluster is the consistency boundary for identity. You never mutate a member's cluster membership outside a cluster transaction — that's how you avoid two golden records claiming the same source row.
  • GoldenRecord is the consistency boundary for truth. Survivorship runs atomically per entity; a half-applied survivor set must never be observable. Delta MERGE + a single commit guarantees this.
  • StewardshipCase is the consistency boundary for human decisions. A merge approval and the resulting golden-record change are linked by case_id so the audit trail is reconstructable.
  • Temporal rule: GoldenRecord is bitemporal — version + valid_from/valid_to + Delta time travel. You can answer "what did we believe about this customer on 2026-01-01?" without a separate history store.

Domain events & data contracts

Why CDF is the event bus

In a native MDM you don't stand up Kafka to move events between contexts inside the lakehouse — Change Data Feed on each context's tables is the event log. It's durable, replayable, governed by UC, and gives every consumer exact incremental deltas with offsets. External systems can still get Kafka via a publishing adapter, but intra-product choreography is pure Delta.

Data contracts (the seams)

Each arrow in the context map is a contract: a named, versioned schema with explicit nullability, semantics, and an owner. Contracts are enforced as DLT expectations on the producing side and validated on read by the consumer. Breaking a contract is a versioned, reviewed change — not a silent column rename. This is what lets nine teams own nine contexts without a coordination meeting per release.
